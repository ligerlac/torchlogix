# TorchLogix: Direct HDL Generation Feature

## Summary

Successfully implemented direct Verilog generation for TorchLogix models, enabling comparison between the traditional C→HLS→RTL pipeline and direct HDL synthesis.

## What Was Implemented

### 1. Core HDL Generation Infrastructure
**File**: `src/torchlogix/hdl_generator.py` (NEW)
- Complete mapping of all 16 Boolean gate operations to Verilog syntax
- Module template generation utilities
- Wire naming and formatting helpers
- Truth table validation support

**Key Features**:
- `gate_id_to_verilog()`: Maps gate IDs (0-15) to Verilog expressions
- `generate_verilog_module()`: Creates complete Verilog module templates
- Support for all 16 Boolean operations: AND, OR, XOR, NAND, NOR, XNOR, constants, passthroughs, implications

### 2. Extended CompiledLogicNet Class
**File**: `src/torchlogix/compiled_model.py` (MODIFIED)

**New Methods**:
- `get_gate_verilog()`: Generate Verilog for individual gate operations
- `_get_linear_layer_verilog()`: Generate Verilog for LogicDense layers
- `_get_conv_layer_verilog()`: Generate Verilog for LogicConv2d/3d layers (binary tree structure)
- `get_verilog_code()`: Generate complete Verilog module for entire network
- `export_hdl()`: Export Verilog to file

### 3. Example Script
**File**: `examples/export_to_verilog.py` (NEW)

Demonstrates:
- Creating a simple LogicDense network
- Optional training
- Generating Verilog HDL
- Comparing with C code generation
- Testing functional equivalence

### 4. Test Suite
**File**: `tests/test_verilog_generation.py` (NEW)

Tests for:
- Gate mapping correctness
- Verilog generation for simple and multi-layer models
- File export functionality
- Truth table validation

## Usage Example

```python
import torch
import torch.nn as nn
from torchlogix.layers import LogicDense
from torchlogix import CompiledLogicNet

# Create model
model = nn.Sequential(
    LogicDense(8, 4, connections="random"),
    LogicDense(4, 2, connections="random")
)

# Compile and generate Verilog
compiled = CompiledLogicNet(model)
verilog_code = compiled.get_verilog_code(module_name="my_logic_net")

# Export to file
compiled.export_hdl("./output", format="verilog")
```

## Generated Verilog Structure

For a simple 2-layer network (4→2 neurons), the generated Verilog looks like:

```verilog
module test_net (
    input wire [3:0] inp,
    output wire [1:0] out
);

// Generated by TorchLogix
// Input size: 4 bits
// Output size: 2 bits

// Layer 0: LogicDense (2 neurons)
    assign out[0] = inp[2];
    assign out[1] = inp[3];

endmodule
```

For more complex networks with multiple gates:

```verilog
module logic_net (
    input wire [7:0] inp,
    output wire [3:0] out
);

// Layer 0: LogicDense (4 neurons)
    assign layer_0_out[0] = (inp[0] & inp[2]);  // AND gate
    assign layer_0_out[1] = (inp[1] | inp[3]);  // OR gate
    assign layer_0_out[2] = (inp[4] ^ inp[5]);  // XOR gate
    assign layer_0_out[3] = ~(inp[6] & inp[7]); // NAND gate

// Layer 1: LogicDense (2 neurons)
    assign out[0] = (layer_0_out[0] | layer_0_out[1]);
    assign out[1] = (layer_0_out[2] ^ layer_0_out[3]);

endmodule
```

## Design Decisions

### Why Direct Gate Synthesis (Not LUTs)?

1. **Natural mapping**: TorchLogix already defines 16 gate operations
2. **Better optimization**: Modern synthesis tools optimize gate-level HDL better
3. **More readable**: `assign out = a & b;` vs large case statements
4. **Binary structure**: Each neuron has exactly 2 inputs → perfect for direct gates

### Comparison with LogicNets

| Aspect | LogicNets | TorchLogix |
|--------|-----------|-----------|
| **Approach** | Truth tables → LUTs | Gate IDs → Direct operators |
| **Intermediate** | Enumerated I/O mappings | Selected gate operations |
| **Neurons** | Multi-input (configurable fan-in) | 2-input binary gates |
| **Use Case** | Sparse quantized NNs | Logic gate NNs |

## Benefits

1. **Direct comparison**: Compare C→HLS vs direct Verilog approaches
2. **Validation**: Verify that gate-level logic matches expectations
3. **Optimization analysis**: Understand synthesis tool behavior
4. **Alternative deployment**: Skip HLS for simple networks
